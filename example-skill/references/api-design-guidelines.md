# API 설계 가이드라인

## RESTful API 원칙

### 리소스 명명
- 복수형 사용: `/api/v1/users` (O), `/api/v1/user` (X)
- 소문자와 하이픈 사용: `/api/v1/user-profiles`
- 계층 구조 명확히: `/api/v1/users/{id}/posts`

### HTTP 메서드
- GET: 리소스 조회
- POST: 새 리소스 생성
- PUT: 리소스 전체 업데이트
- PATCH: 리소스 부분 업데이트
- DELETE: 리소스 삭제

### 상태 코드
- 200: 성공
- 201: 생성 성공
- 400: 잘못된 요청
- 401: 인증 필요
- 403: 권한 없음
- 404: 리소스 없음
- 500: 서버 오류

## 버전 관리
- URL에 버전 포함: `/api/v1/`, `/api/v2/`
- 하위 호환성 유지
- 명확한 변경 로그 제공
